Key Findings
⚡
~50% latency reduction with gRPC gRPC/Protobuf reduces average latency by 47–51% vs REST/JSON across all message sizes due to binary serialization and HTTP/2 multiplexing.
📈
1.7–2.2× throughput improvement gRPC achieves 372 rps vs REST's 197 rps on small messages (single connection). The advantage grows under higher concurrency.
🔧
REST degrades faster under load At c=50, REST latency increases 2.2× from baseline. gRPC degrades only 1.5× — HTTP/2 stream multiplexing handles backpressure better.
📦
Large message advantage grows For 64KB payloads, gRPC's binary Protobuf encoding is ~4× more compact than JSON, reducing serialization time and network bytes.
📊
Tail latency (P99) is the real story REST P99 at large messages is 137ms vs gRPC's 72ms — a 47% reduction. Tail latency drives user-perceived slowness in real systems.
🚀
Next steps: shared memory / Unix sockets For intra-host services, Unix domain sockets can reduce latency to <0.5ms, eliminating TCP stack overhead entirely.
